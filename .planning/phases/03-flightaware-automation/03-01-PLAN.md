Create `supabase/migrations/052_sync_flightaware_daily_fn.sql` with a PL/pgSQL function `sync_flightaware_daily()` that:

1. Queries all airport ICAO codes:
   ```sql
   FOR r IN SELECT codigo_icao FROM aeroporto WHERE codigo_icao IS NOT NULL AND codigo_icao != '' LOOP
   ```

2. For each airport, fetches yesterday AND today (two calls per airport):
   - yesterday: `date_from = (now() - interval '1 day')::date::text || 'T00:00:00Z'`, `date_to = (now() - interval '1 day')::date::text || 'T23:59:59Z'`
   - today: `date_from = now()::date::text || 'T00:00:00Z'`, `date_to = now()::date::text || 'T23:59:59Z'`

3. Calls `fetch_fr24(airport_icao, date_from, date_to)` for each day. If result contains `error` key, logs and continues (RAISE NOTICE).

4. Iterates over the `data` array in the result and upserts each flight into `cache_voo_f_r24`:
   ```sql
   INSERT INTO cache_voo_f_r24 (fr24_id, numero_voo, airport_icao, data_voo, data_expiracao, status, raw_data, updated_date, created_date)
   VALUES (
     flight->>'fr24_id',
     COALESCE(flight->>'flight', ''),
     flight->>'airport_icao',
     COALESCE((flight->>'datetime_landed'), (flight->>'datetime_takeoff'), now()::text)::date,
     (now() + interval '30 days')::date,
     'pendente',
     flight,
     now(),
     now()
   )
   ON CONFLICT (fr24_id) DO UPDATE SET
     raw_data = EXCLUDED.raw_data,
     numero_voo = EXCLUDED.numero_voo,
     data_voo = EXCLUDED.data_voo,
     updated_date = now()
   WHERE cache_voo_f_r24.status != 'importado';
   ```

5. Skips entries where `fr24_id` is null or empty.

6. Returns a summary JSONB: `{ "airports_processed": N, "flights_upserted": N, "errors": [...] }`

Function signature:
```sql

RETURNS jsonb
LANGUAGE plpgsql
SECURITY DEFINER
AS $fn$ ... $fn$;
```

Create `supabase/migrations/053_schedule_flightaware_daily.sql` that:

1. Enables pg_cron extension (idempotent):
   ```sql
   CREATE EXTENSION IF NOT EXISTS pg_cron;
   GRANT USAGE ON SCHEMA cron TO postgres;
   ```

2. Removes any existing job with the same name before re-adding (idempotent):
   ```sql
   SELECT cron.unschedule('flightaware-daily-sync') WHERE EXISTS (
     SELECT 1 FROM cron.job WHERE jobname = 'flightaware-daily-sync'
   );
   ```
   NOTE: The above pattern may not work in all Postgres versions. Use this safer pattern instead:
   ```sql
   DO $$
   BEGIN
     IF EXISTS (SELECT 1 FROM cron.job WHERE jobname = 'flightaware-daily-sync') THEN
       PERFORM cron.unschedule('flightaware-daily-sync');
     END IF;
   END $$;
   ```

3. Schedules the job:
   ```sql
   SELECT cron.schedule(
     'flightaware-daily-sync',     -- job name
     '0 3 * * *',                  -- 03:00 UTC every day
     $$SELECT sync_flightaware_daily()$$
   );
   ```

4. Add a comment block at the top:
   ```sql

   -- To manually trigger: SELECT sync_flightaware_daily();
   ```
