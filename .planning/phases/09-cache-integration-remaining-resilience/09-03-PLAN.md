App.jsx already contains:
1. An ErrorBoundary class component (lines 27-95) that:
   - Catches ChunkLoadError via message pattern: "Failed to fetch dynamically imported module" or "Importing a module script failed"
   - Auto-reloads once (within 30s cooldown tracked in sessionStorage 'chunk_reload_at')
   - Shows spinner during auto-reload
   - Shows manual "Tentar novamente" button if cooldown active
   - Clears cooldown after 15s of successful render
   - Reports to Sentry via captureException

2. The ErrorBoundary placement in App() render:
   ```jsx
   <ErrorBoundary>
     <Suspense fallback={<PageLoader />}>
       <Routes>
         <Route path="/login" element={<Login />} />
         {PublicPages.map(...)}
         <Route path="*" element={<AuthenticatedApp />} />
       </Routes>
     </Suspense>
   </ErrorBoundary>
   ```

3. AuthenticatedApp also has its own Suspense (line 137):
   ```jsx
   <Suspense fallback={<PageLoader />}>
     <Routes>
       {Object.entries(Pages).map(...)}  // all lazy pages from pagesConfig
     </Routes>
   </Suspense>
   ```

Gap to check: The AuthenticatedApp Suspense (line 137) is NOT wrapped in an ErrorBoundary. If a ChunkLoadError happens on a lazy page load AFTER authentication (most common scenario), the outer ErrorBoundary in App() should still catch it because React error boundaries propagate up the tree. Verify this is correct.

Second gap to check: The 'chunk_reload_at' cooldown uses sessionStorage.getItem in componentDidCatch but the 15s clear timer is set in constructor. Verify the constructor timer fires correctly (it does — componentWillUnmount clears it via clearTimeout).
