Import pattern (same as 04-02/04-03):
```javascript

```

Key namespaces relevant to these components:
- `btn.*` — save, cancel, delete, edit, add, search, filter, export, import, back, close, refresh, new, view, download, upload, submit, approve, reject, print, loading, processing
- `label.*` — name, email, phone, status, date, description, category, priority, airport, company, actions, observations, type, total, results, no_results, search_placeholder, all, active, inactive, created_at, updated_at
- `msg.*` — confirm_delete, saved_success, deleted_success, error_occurred, loading, no_data
- `tab.*` — flights, config, reports, history, panel, list
- `page.documentos.*` — document page strings
- `page.grf.*` — GRF strings
- `page.config_tarifas.*` — tariff config strings
- `page.fundo_maneio.*` — financial strings
- `safety.*` — Safety domain
- `inspecoes.*` — Inspections domain
- `manutencao.*` — Maintenance domain
- `flightaware.*` — FlightAware strings
- `general.*` — error, success, loading, save, cancel, delete, edit, close, add

Note: `configuracoes/ZAPI*` files are excluded from this plan — they handle ZAPI (WhatsApp) integration and should only be translated if ZAPI is actively used. Defer to avoid breaking untested config flows.

Note: `lib/*` files (aeroportosAngola, auditoria, export, notificacoes, tariffCache, etc.) are utility/logic files — skip them unless they render JSX with user-visible strings.

Note: `ui/*` components are Radix UI primitives — they must NOT have i18n added (generic components).
