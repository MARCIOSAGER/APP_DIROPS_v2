<objective>
Add minimum column width constraints to VoosTable and Proforma table so they are readable and scrollable on standard 1280px laptop screens without columns collapsing.

Purpose: At typical laptop widths, the VoosTable has 11 columns with no minimum widths, causing narrow columns (e.g. "Tipo", "Horários") to collapse to near-unreadable sizes. Proforma table suffers the same. Adding explicit `min-w-[Npx]` via Tailwind to `<TableHead>` elements ensures the overflow-x-auto container kicks in cleanly.

Output: Both tables have column-level minimum widths, overflow-x-auto already present keeps them scrollable, no layout regressions.
</objective>
